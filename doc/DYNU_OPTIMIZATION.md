# Dynu API 优化说明

## 问题
Dynu API 请求经常失败，影响用户体验。

## 优化措施

### 1. 减少超时时间
- **之前**: 30 秒超时
- **现在**: 15 秒超时
- **原因**: 30 秒太长，用户等待时间过久。15 秒足够处理正常请求，同时能更快发现问题。

### 2. 添加自动重试机制
- **重试次数**: 最多 3 次（共 4 次尝试）
- **退避策略**: 指数退避（1秒、2秒、4秒）
- **智能重试**: 
  - 4xx 客户端错误（除 429 限流）不重试
  - 5xx 服务器错误会重试
  - 网络错误会重试
  - 429 限流错误会重试

### 3. 连接池优化
```go
Transport: &http.Transport{
    MaxIdleConns:        10,
    MaxIdleConnsPerHost: 5,
    IdleConnTimeout:     30 * time.Second,
    DisableKeepAlives:   false,
}
```
- 保持连接复用，减少 TCP 握手开销
- 限制空闲连接数，避免资源浪费

### 4. 更好的错误处理
- 自定义 `HTTPError` 类型，包含状态码和详细信息
- 区分不同类型的错误，便于调试
- 添加 User-Agent 标识

## 预期效果
1. 请求成功率提升
2. 失败时自动重试，减少手动操作
3. 更快的响应时间
4. 更好的错误提示

## 使用建议
- 如果仍然遇到频繁失败，检查 API Key 是否正确
- 检查网络连接是否稳定
- 查看 Dynu 服务状态: https://status.dynu.com/
